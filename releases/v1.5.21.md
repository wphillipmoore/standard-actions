## Documentation

- **add design spec for extracting workflow logic into composite actions**
Covers issue #436 — documents the approved design for three new composite actions (validate-inputs, registry-publish, freeze-internal-refs) and a modification to docs-deploy for mike command auto-detection.

- **add permissions note to registry-publish spec**
- **apply pushback review to extraction spec (#436)**
Addresses five findings from spec review:
- Require env: over expression interpolation in shell steps (security)
- Add supported-language validation (python, java, ruby, rust, go)
- Unify publish path — single flow with conditional credential guard
- Restrict freeze-internal-refs sed to uses: lines only
- Document publish-command input name mapping, defer namespace to #440

- **add implementation plan for extracting workflow logic (#436)**
- **decouple build audits from registry-publish gate (#436)**
Build/attestation/SBOM pipeline now runs for all supported build languages
(python, java, ruby, rust, go) on new tags, regardless of registry-publish
flag. The registry-publish flag only controls the final push-to-registry
step. This ensures Go (and any language) gets the same audit bar as
languages with explicit registry publish commands.

- **add alignment review for workflow extraction (#436)**
Spec-to-plan alignment verified: 29 requirements covered, 8 tasks in scope,
6 design decisions reflected. Two issues found and resolved during review:
- eval pattern documented in spec security constraint
- Build audits decoupled from registry-publish gate (separate commit)
