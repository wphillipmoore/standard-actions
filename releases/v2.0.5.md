
# Release 2.0.5 (2026-05-19)

## Chores

- **bump version to 2.0.5**
- **remove redundant vergil-tooling key from vergil.toml**
vergil-tooling consolidated the [dependencies] section so only the vergil key is required (vergil-tooling#755). Ref #485

- **remove per-repo templates in favor of org defaults**
Issue templates and PR template are now inherited from vergil-project/.github. Per-repo copies were byte-identical and are no longer needed.

- **remove co-authors section, normalize vergil dep to v2.0**

## Features

- **deploy permission model configuration**
- **deploy deny rules to project-level settings**
- **replace title-scanning with deterministic tracking-issue lookup**
Remove the tracking-issue input and title-scanning fallback. The action
now calls vrg-resolve-tracking-issue to deterministically extract the
release tracking issue from the merge commit on main. If the tool fails,
the action fails — no PR is created without issue linkage.

Convert the develop version idempotency check from a silent skip to a
hard assertion. The upstream CI version-divergence gate should prevent
this condition; if develop already has the bumped version, something has
gone wrong and the workflow stops with a diagnostic error.

Remove the bump-needed output (the action now always creates a PR on
success). Update docs to reflect current inputs and behavior.
