
# Release 1.2.1 (2026-04-24)

## Bug fixes

- **use env vars + quoted heredoc in version-bump-pr Update step**
Surfaced on wphillipmoore/standard-tooling-plugin#78 (v1.4.1
publish): version-regex `(\s*"version":\s*").*("\s*)` contains
literal double-quotes; GitHub expression interpolation put the
raw value into the bash `python3 -c "..."` string, bash's outer
double-quote closed at the first embedded `"`, the `(` in the
regex became a subshell-open, and the `)` produced
"unexpected token" at exit.

Latent since the step was written. Only surfaced once earlier
composite failures (CHANGELOG merge, missing version-replacement)
stopped masking it.

## Fix

Pass values via `env:` and use a single-quoted heredoc delimiter
`<<'PYEOF'` so bash performs zero interpolation on the body. The
regex, replacement, and next version are all read from
`os.environ` inside the python block.

No caller-visible change to the composite's interface (same
inputs, same behavior). Patch-level bump — v1.2.0 consumers
automatically get the fix when `v1.2` rolls forward.
