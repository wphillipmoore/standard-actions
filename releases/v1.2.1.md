
# Release 1.2.0 (2026-04-24)

## Features

- **drop auto-merge from version-bump-pr composite (bump 1.2.0) (#193)**
Step 2 of 3 for the poll-and-merge rework (plugin#57).

The composite used to end with:

    gh pr merge --auto --merge --delete-branch <branch>

That call has been silently failing on every publish run since
auto-merge was disabled fleet-wide — the bump PR got created, the
step threw, the workflow was marked failure, and the user merged
manually anyway.

Drop it. `pr-url` is already an output; the release skill drives
the merge via st-merge-when-green once CI is green.

Action name / description, in-file comment, and public docs all
updated to match. No other auto-merge references in the repo.

VERSION 1.1.4 -> 1.2.0. Minor bump because behavior changes for
consumers pinned to `@v1.1`; the rolling tag will not move. They
opt in to `@v1.2` alongside a coordinated skill rewrite.
