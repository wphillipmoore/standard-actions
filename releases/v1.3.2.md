
# Release 1.3.2 (2026-04-27)

## Bug fixes

- **use Issues API instead of Search API for tracking-issue lookup**
The tracking-issue lookup used 'gh issue list --search ... in:title', which hits GitHub's Search API. That API has indexing latency: an issue created seconds earlier is typically not yet indexed, so the search returns empty and the bump PR ships with no Ref #N linkage, failing st-pr-issue-linkage downstream.

Switch to 'gh issue list' (Issues API, real-time) with a client-side title filter via --jq. The tracking issue is found immediately.
