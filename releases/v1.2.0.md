
# Release 1.1.3 (2026-04-23)

## Bug fixes

- **pin markdownlint-cli@0.41.0, add run-codeql flag (#172) (#173)**
- **rename dev-docs container references to dev-base (#182)**
Update reusable workflows and standards-compliance action to use dev-base:latest. The dev-docs image was deleted as part of the dev-docs to dev-base rename. Ref #181

- **stabilize version-bump-pr back-merge and docs-deploy container cwd (#188)**
Two pipeline-stability fixes rolled into one change; both block
end-to-end release automation and were caught during the
standard-tooling-plugin v1.3.0/v1.3.1 first-real-release on
2026-04-23.


## CI

- **remove standard-tooling setup hacks from standards-compliance (#179)**
* ci: remove standard-tooling setup hacks from standards-compliance (#214)

Now that standard-tooling is pre-installed in the dev-docs container
image, the standards-compliance action no longer needs to clone
standard-tooling, add it to PATH, install Node.js, or install
markdownlint-cli. Replace all setup steps with a hard requirement
for st-repo-profile and add dev-docs container to CI jobs.


## Features

- **add shared publish-release reusable workflow (#163)**
Shared workflow_call reusable workflow that eliminates publish.yml
duplication across all 6 mq-rest-admin repos. Uses an ecosystem
selector (python|rust|ruby|go|java|none) to drive conditional
setup-* action steps, with all other ecosystem-specific behavior
passed as shell-command string inputs.

Fixes audit findings:
- Normalizes SBOM output to dist/ (was root for Go, target/ for Java)
- Adds credential guard for Rust/Ruby/Java registry publish
- Consistent action refs via @develop
- Consistent step ordering: publish before tag-and-release

- **make docs-deploy container-aware (#174)**
* feat: make docs-deploy install steps container-aware

Add a detect step that checks if mike is already on PATH. Gate Python
setup, mkdocs/mike install, and pyyaml install on mike not being
pre-installed. This allows the action to work both with and without the
dev-docs container.

- **adopt git worktree convention for parallel AI agent development (#184)**
Applies Phase 3 of the worktree-convention rollout from
wphillipmoore/standard-tooling#258.

Adds `.worktrees/` to .gitignore and a Parallel AI agent development
section to CLAUDE.md describing the structure, rules, and canonical
Agent prompt contract. Links to the canonical spec in standard-tooling.
