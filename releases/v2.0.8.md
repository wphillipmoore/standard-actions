
# Release 2.0.8 (2026-05-20)

## Bug fixes

- **align CLAUDE.md with canonical consumer template**
- **add dev-base container to github-config job**
The ops-github-config workflow ran on bare ubuntu-latest without a container image, so uv was not available for the vergil-tooling setup action. All other reusable workflows use the dev-base container which has uv pre-installed.

- **replace hardcoded container images with input-driven prefix**
The ops-github-config and cd workflows hardcoded their container image
references instead of using the container-prefix input pattern. This
prevented consumers from overriding the image prefix when validating
dev Docker image changes.

- ops-github-config.yml: add container-prefix input, replace hardcoded
  dev-base with the standard input-driven pattern
- cd.yml: add container-prefix workflow_dispatch input, replace
  hardcoded prod-base, propagate prefix to cd-docs.yml call
- Document the container prefix override mechanism in the reusable
  workflows docs

- **revert cd.yml overrides and update container prefix docs**
Remove the workflow_dispatch input and pass-through from cd.yml — the
override mechanism is editing the consumer workflow files, not runtime
inputs. The cd.yml self-referencing file correctly hardcodes prod-base.

Update the container prefix documentation to explain both use cases:
local validation via vrg-docker-run, and full end-to-end CI/CD testing
by committing container-prefix overrides to consumer workflow files.

- **correct container prefix docs for local validation**
vrg-docker-run reads the image prefix from vergil.toml [docker]
image-prefix, defaulting to prod. Document this as the local override
mechanism instead of the incorrect claim that it defaults to dev.

- **replace vergil.toml prefix instructions with vrg-docker-run reference**
The vergil.toml docker image-prefix setting is being removed in favor of
a CLI flag on vrg-docker-run (vergil-project/vergil-tooling#883). Replace
the vergil.toml instructions with a reference to the vrg-docker-run
documentation for local validation with dev containers.

- **correct vrg-docker-run documentation URL**
Point to the actual CLI tools overview page in the vergil-tooling docs
site rather than a fabricated URL path.
