## Refactoring

- **auto-derive release title and notes from repo metadata**
Make release-title and release-notes optional with sensible defaults. Title defaults to repo-name + tag, notes default to a link to the GitHub Pages docs site. Callers that don't need custom values can omit both inputs.

- **remove domain-specific checkout-common inputs**
Remove checkout-common and checkout-common-ref inputs and the mq-rest-admin-common checkout step. Domain-specific logic violates design goal of keeping standard-actions generic. The mq-rest-admin family uses pre-deploy-command in the reusable workflow instead.

- **use st-version bump instead of regex substitution**
Replace caller-provided version-file, version-regex, version-replacement, version-regex-multiline, and develop-version-command inputs with st-version which reads standard-tooling.toml and knows per-language conventions. Uses a temporary worktree to check develop version. st-version bump handles both version file edit and lockfile maintenance.

- **derive all inputs from standard-tooling.toml, zero required caller inputs**
Remove all required caller inputs (ecosystem, version-command, version-file, version-regex, release-title, release-notes, and per-language version inputs). The workflow detects ecosystem from standard-tooling.toml, uses st-version for version operations, and derives build/registry commands per ecosystem. Inner job name changed from 'publish: release' to 'release' for the check name convention.
