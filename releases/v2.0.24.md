
# Release 2.0.24 (2026-05-28)

## Bug fixes

- **install uv via setup-uv when not present on bare runners**
The vergil-tooling setup action assumed uv was pre-installed, which
breaks on bare ubuntu-latest runners used by CD pipelines. Add a
conditional astral-sh/setup-uv step that only runs when uv is not
found on PATH.

Replace duplicated vergil-tooling install logic in the Trivy action
with delegation to the setup action, establishing a single source of
truth for the install path.

- **use astral-sh/setup-uv@v7 (v8 has no rolling major tag)**
- **use pip to install uv on bare runners instead of astral-sh/setup-uv**
Replace astral-sh/setup-uv with pip install to avoid a circular
dependency: the repo allowlist compliance check rejects astral-sh/*
until vergil-tooling is updated, but vergil-tooling can't be fixed
until this action works on bare runners.

The pip approach matches how the Docker base images install uv and
has no external action dependency. A follow-up issue (#643) tracks
upgrading to astral-sh/setup-uv once the allowlist and compliance
tooling are updated.
