
# Release 2.0.20 (2026-05-27)

## Bug fixes

- **map go to p/golang and capture scan exit codes**
The Semgrep registry uses p/golang, not p/go (HTTP 404). Added a language-name mapping so callers passing language: go get the correct ruleset. Also capture semgrep scan exit codes instead of letting bash -e kill the step, so the SARIF analysis step can provide diagnostics when the scan fails.
