
# Release 1.5.3 (2026-05-05)

## Bug fixes

- **remove || true from yamllint in ci-quality.yml**
yamllint errors were silently suppressed, making the CI gate decorative. This restores enforcement to match st-validate-local parity. Fixes #335

- **remove || true from markdownlint, hadolint, and actionlint**
Same class of bug as yamllint: linter failures were silently suppressed, making CI gates decorative. All four linters in ci-quality.yml now correctly fail the workflow on errors.

- **fix linter violations exposed by removing || true**
Add canonical .markdownlint.yaml config (parity with st-validate-local). Add .markdownlintignore for generated content (releases, changelogs, paad). Fix markdownlint violations in AGENTS.md, CLAUDE.md, and docs/specs. Fix yamllint line-length violations in docs-deploy and standards-compliance actions.

- **fix markdownlint violations in CI workflow reset design spec**
- **add safe.directory for container-based CI in version-divergence action**
- **add safe.directory to setup action for container-based CI**
- **always install standard-tooling from pinned tag, never skip**
