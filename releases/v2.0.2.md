
# Release 2.0.2 (2026-05-13)

## Bug fixes

- **use full action paths in reusable workflows and rename st-* to vrg-***
Reusable workflows referenced local actions with ./ which resolves
to the caller repo, not vergil-actions. Changed to full paths.
Also renamed remaining st-* command references to vrg-*.

- **use full action path in standards-compliance composite action**
- **use release/* branch prefix in version-bump-pr (#468)**
* fix(cd): use release/* branch prefix in version-bump-pr action

Changes branch naming from chore/bump-version-<next> to
release/bump-version-<next> so vrg-merge-when-green accepts
automated bump PRs.

Also fixes vergil.toml dependency key name from 'vergil' to
'vergil-tooling' to match what vrg-docker-run expects.

- **migrate from app-id to client-id for GitHub App token (#469)**
* fix(cd): migrate from app-id to client-id for GitHub App token

Switches create-github-app-token from the deprecated app-id parameter
to client-id, which uses the OAuth Client ID instead of the numeric
App ID. This eliminates the deprecation warning and may resolve the
JWT decode failure observed during vergil-tooling v2.0.2 publish.

The reusable workflow secret is renamed from APP_ID to APP_CLIENT_ID.
Callers of cd-release.yml must update their secret passthrough.
