
# Release 2.0.21 (2026-05-27)

## Bug fixes

- **handle missing primary-language in vergil.toml**
Use .get() with empty-string fallback instead of hard subscript when reading primary-language from vergil.toml. The key was removed in PR #609 but the docs deploy action still read it with a bare subscript, causing a KeyError in downstream CD pipelines. Ref #617

- **revert internal refs to @develop and replace trivy shell with vrg-trivy-scan**
All internal action references in reusable workflows were incorrectly frozen to @v2.0 on the develop branch. This broke self-referencing CI: changes merged to develop were never tested because workflows referenced old tagged code instead of the current branch. Reverted all 18 @v2.0 refs back to @develop so vrg-freeze-refs can properly rewrite them to exact patch versions at release time. Also replaced 80 lines of nested-shell-in-Docker trivy scan orchestration with vrg-trivy-scan, added Install vergil-tooling steps to codeql/trivy/semgrep jobs that were missing them. Ref #607, Ref #613

- **use local action paths for true self-referencing CI**
Replace all @develop action refs in reusable workflows with ./actions/ local paths. GitHub resolves ./ relative to the workflow file at the ref it was loaded from, so: during PR CI the actions come from the PR branch (true self-referencing), and at release time vrg-freeze-refs rewrites them to absolute tagged refs for downstream consumers. Ref #607, Ref #613

- **run trivy job in prod-base container**
Run the trivy CI job in the standard prod-base container instead of bare ubuntu-latest. This provides uv and other tooling needed by the Install vergil-tooling step. Ref #607

- **revert trivy action to original inline Docker approach**
Revert the trivy action to its original inline Docker implementation. The vrg-trivy-scan tool incorrectly wraps trivy in docker run rather than calling it directly, which needs to be fixed in vergil-tooling before the action can use it. See vergil-tooling#1246. Ref #607

- **use dev container prefix for CI**
Switch CI to use dev-base container images so we can test with updated container images before they are promoted to prod. Ref #607

- **eliminate shell injection patterns in action YAML files**
Move all expression interpolations out of run: script bodies and into env: blocks, referencing them as shell variables instead. This prevents expression injection where crafted input values could execute arbitrary commands during template expansion. Also fixes a bug in docs/deploy where the mike-command resolution step was referencing its own output instead of the input.

- **replace Docker orchestration with vrg-trivy-scan direct invocation**
Replace docker run wrapping in the trivy action with calls to vrg-trivy-scan for fs/image scans and direct trivy CLI for SBOM generation. The trivy job now runs in the container where trivy and vergil-tooling are available. Removes trivy-image and scanners inputs (no longer needed). Also converts the registry-publish trivy reference from @develop to local path.


## Chores

- **remove invalid primary-language from vergil.toml**
Shell is no longer a valid value for primary-language, which is now
only specified for the five main programming languages.

- **remove deprecated version-bump-pr action**
Remove the version-bump-pr composite action and all references to it. This action was deprecated in favor of the vrg-release orchestrator and is no longer referenced by any workflow. Removes the action directory, its docs page, and updates CLAUDE.md, mkdocs nav, action index, and configuration docs. Ref #603


## Refactoring

- **replace inline shell with vrg-semgrep-scan**
Replace 70 lines of embedded shell (ruleset resolution, scan execution, jq-based SARIF evaluation) with a single call to vrg-semgrep-scan from vergil-tooling. The tool handles language-to-ruleset mapping, scan execution with structured exit codes, SARIF parsing, finding evaluation, and GitHub Actions job summaries. The action retains Dockerfile/workflow auto-detection as lightweight shell checks that set --has-dockerfiles and --has-workflows flags. Ref #597

- **replace inline SARIF evaluation with vrg-sarif-evaluate**
Replace 27 lines of jq-based SARIF evaluation shell (nullglob, slurp, filtering, gating) with a single call to vrg-sarif-evaluate from vergil-tooling. The tool handles directory-mode SARIF parsing, severity filtering, structured annotations, and job summaries. Ref #598

- **replace inlined Python scripts with vrg-docs-stage and vrg-docs-patch-nav**
Replace two inlined Python heredoc scripts (release notes staging with semver-sorted index generation, and mkdocs.yml nav patching) with calls to vrg-docs-stage and vrg-docs-patch-nav from vergil-tooling. Eliminates the container path workaround from issue #204. Ref #599

- **replace case-statement command derivation with vrg-ecosystem-resolve**
Replace three duplicated case blocks (build command, publish command, credential secret) with a single call to vrg-ecosystem-resolve from vergil-tooling. The tool writes shell-compatible strings directly to GITHUB_OUTPUT with the correct key names. User overrides append after the tool call (last write wins). Ref #600

- **replace inline validation with vrg-release-validate-inputs**
Replace interdependent conditionals, loop-based language validation, and duplicated supported-languages list with a single call to vrg-release-validate-inputs from vergil-tooling. The tool shares the canonical language list with vrg-ecosystem-resolve. Ref #601

- **replace inline sed/grep logic with vrg-freeze-refs**
Replace two shell steps (sed-based regex freezing with temp file cleanup, and duplicated find/grep validation) with a single vrg-freeze-refs call from vergil-tooling. The tool handles YAML file collection, reference freezing, and unfrozen-ref validation in one pass. Ref #602

- **remove redundant auto-close check already handled by vrg-pr-issue-linkage**
The vrg-pr-issue-linkage tool already validates both issue linkage and auto-close keyword rejection. The inline shell step duplicated the auto-close check with jq extraction and grep regex. Ref #604

- **replace inline comparison with vrg-version-divergence**
Replace 22 lines of version comparison shell (conditional extraction, string comparison, multi-path output writes) with a single call to vrg-version-divergence from vergil-tooling. The tool handles version comparison logic, error messages, and job summaries. The action retains version extraction via user-provided commands and maps the exit code to the diverged boolean output. Ref #605

- **replace sed TOML parsing with python3 tomllib**
Replace the fragile sed regex that extracts the vergil version from vergil.toml with a python3 one-liner using the stdlib tomllib module. The sed approach could not handle comments, multi-line values, or quoted strings correctly. The vrg-resolve-tooling-version tool exists in vergil-tooling but cannot be used here due to a bootstrapping constraint: this action is what installs vergil-tooling, so the tool is not yet available when the action runs. Ref #606
